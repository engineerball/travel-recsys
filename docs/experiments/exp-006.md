# Exp-006 — Scale behavior dataset to 5000 users (2026-05-19)

## Changes from [Exp-005](exp-005.md)

| Change | Where | Detail |
|---|---|---|
| `num_users: 1000 → 5000` | `configs/config.yaml` | Bootstrap-augmented via `augment_user_profiles` (age ±5, province resample, 20% country reassign) |
| Regenerated behavior data | `data/generated/` | 324,237 interactions across 5,000 users (vs 64,666 / 1,000) |

**No model or behavior generator changes.**

**Motivation:** Exp-005 investigation revealed the Exp-004 behavior dataset was ~7x larger (implied ~442k total interactions). When we regenerated for Exp-005, the smaller 64k dataset left accommodation items with only ~8 train interactions each on average (vs ~56 at Exp-004 scale). Scaling to 5,000 users restores dataset density and makes the Exp-005 → Exp-006 comparison between experiments on equivalent eval sets (31.7k val, vs 6k in Exp-005).

Files changed: `configs/config.yaml`, `data/generated/` (regenerated).

## Notes

- **Accommodation recovered** — Hit@10 0.166 → 0.179, confirming dataset size was the root cause of the Exp-005 regression, not the article behavior changes.
- **Overall / event / article numbers appear lower** — artifact of larger, harder val set (31.7k vs 6k). At this scale the FAISS index is also larger (ATTRACTION 2,673 vs 1,912, ACCOMMODATION 1,472 vs 1,030, EVENT 1,639 vs 492), increasing retrieval difficulty.
- FAISS index now contains 1,639 events (vs 492 in Exp-005). More distractor items means lower Hit@K at fixed K — this is expected and healthy for a retrieval system.
- **Exp-006 is now the stable baseline** at 5k users / ~32k val. All future experiments should compare against these numbers.

## Open issues

- Event Hit@10 (0.067) is the weakest type at this scale. More events pass the `min_item_interactions` threshold now (1,639 vs 492), adding harder distractors.
- Attraction Hit@10 (0.037) still low — text projection may need more epochs or larger warmup to integrate with structured features.
- Consider bumping `steps_per_epoch` or epochs now that training data is 5x larger.
