# Exp-003 — Catalog pruning + real text embeddings (2026-05-19)

## Changes from [Exp-002](exp-002.md)

| Change | Detail |
|---|---|
| Catalog pruning | Drop items with < 3 interactions before training (`min_item_interactions=3` in config) |
| Text embeddings | Pre-computed via Vertex AI `text-embedding-004` (MRL, truncated to 256-dim) for all 4 types; previously zeros |
| Eval fix | `evaluate.py` now filters val interactions to indexed items only — items below prune threshold can never be retrieved, including them inflated miss rate |

**Pruned catalog sizes (training + FAISS index):**

| Type | Raw | Indexed |
|---|---|---|
| Attraction | 8,632 | 2,718 |
| Accommodation | 2,972 | 1,484 |
| Event | 21,376 | 2,123 |
| Article | 1,559 | 1,455 |

Event: 21,376 → 2,123 (kept 10%). 90% of events had ≤2 lifetime interactions — dead catalog weight removed.

## Notes

- **Event** biggest winner — Hit@50 up 42%, Hit@100 up 53%. Pruning gave dense training signal; text embeddings differentiated events within the same category.
- **Attraction** improved across the board; gains modest — structured features (province, subcat, days_open) already carried most signal.
- **Article** flat — pre-existing embeddings already in place; feature set too sparse to differentiate.
- **Eval fix** meaningful mainly for events: 831 of 12,099 event val interactions were for non-indexed items (6.9%).

## Open issues

- Article Hit@10 = 1.9% — feature set too thin (`article_type_id + pub_month`) with no view-count signal.
- Attraction regression risk when text embeddings added — text projection needs enough epochs to integrate with structured features.
